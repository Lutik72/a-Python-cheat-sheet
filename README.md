# Python: Основы (Краткий конспект)

## Переменные и типы данных

### Переменные
**Переменная** — именованная область памяти для хранения данных.

```python
name = "Анна"      # строка
age = 25           # целое число
height = 1.68      # дробное число
```

> 📌 **Правила именования:** латиница, цифры и `_`. Не начинается с цифры, без пробелов, осмысленно, стиль `lower_case_with_underscores`

> ⚠️ **Нельзя использовать ключевые слова:** `False`, `True`, `None`, `and`, `or`, `not`, `if`, `else`, `for`, `while`, `def`, `return` и др.

---

### Базовые типы данных

#### `int` — целые числа
```python
num = 100
negative = -50
big = 7_000_000_000  # подчеркивание для читаемости

# Преобразование
int("123")     # 123 из строки
int(3.14)      # 3 (отбрасывает дробную часть!)
```

#### `float` — дробные числа
```python
pi = 3.14159
temp = -5.5

# Преобразование
float("3.14")  # 3.14 из строки
float(100)     # 100.0
```

#### `str` — строки
```python
name = 'Анна'
message = "Привет"

# F-строки (встраивание переменных)
age = 25
text = f"Мне {age} лет"  # "Мне 25 лет"
```

> 💡 **F-строки** — самый удобный способ форматирования: `f"Текст {переменная} текст"`

---

### Списки (`list`)
Упорядоченная коллекция элементов.

```python
fruits = ['яблоко', 'банан', 'апельсин']

# Основные операции
len(fruits)                # 3 - длина
fruits.append('груша')      # добавить в конец
fruits.insert(1, 'киви')    # вставить на позицию 1
fruits.remove('банан')      # удалить по значению

# Доступ по индексу (с 0)
print(fruits[0])  # первыq элемент
print(fruits[-1]) # последний элемент
```

---

### Словари (`dict`)
Пары "ключ-значение".

```python
pets = {
    'кошка': 'Мурка',
    'собака': 'Барсик',
    'хомяк': 'Пушистик'
}

# Доступ
print(pets['кошка'])        # Мурка
pets['попугай'] = 'Кеша'    # добавление

# Перебор
for key in pets:                    # по ключам
for value in pets.values():          # по значениям
for key, value in pets.items():      # по парам
```

---

### Множества (`set`)
Уникальные элементы, порядок не гарантируется.

```python
fruits = {'яблоко', 'банан', 'апельсин', 'банан'}
print(fruits)  # {'яблоко', 'банан', 'апельсин'} - дубликат удален

# Удаление дубликатов из списка
unique = set(['Москва', 'Париж', 'Москва'])  # {'Москва', 'Париж'}

# Проверка наличия
if 'яблоко' in fruits:
    print("Есть такое!")
```

#### Операции с множествами
```python
a = {1, 2, 3}
b = {3, 4, 5}

a.union(b)           # {1, 2, 3, 4, 5} - объединение
a.intersection(b)    # {3} - пересечение
a.difference(b)      # {1, 2} - разность
```

## Операции с числами

### Арифметические операторы
```python
+   # сложение
-   # вычитание
*   # умножение
/   # деление (возвращает float)
//  # целочисленное деление
%   # остаток от деления
**  # возведение в степень
```
> 📌 **Приоритет:** `()` → `**` → `* / // %` → `+ -`

### Полезные приемы
Алгоритм получения цифр n-значного числа `num`:

* **Вариант 1:**
  * Последняя цифра: `num % 10`
  * Предпоследняя цифра: `(num // 10) % 10`
  * Предпредпоследняя цифра: `(num // 100) % 10`
  * ... и т.д

* **Вариант 2:**
  * Последняя цифра: `num % 10`
  * Предпоследняя цифра: `(num % 100) // 10`
  * Предпредпоследняя цифра: `(num % 1000) // 100`
  * ... и т.д

## Операторы сравнения и логические операторы

### Сравнение
```python
>   # больше
<   # меньше
>=  # больше или равно
<=  # меньше или равно
==  # равно
!=  # не равно
```

### Логические операторы
```python
and  # И (оба истина)
or   # ИЛИ (хотя бы один)
not  # НЕ (отрицание)
```

✅ Таблица истинности для оператора `and`:

|   a   |   b   | a and b |
|:-----:|:-----:|:-------:|
| false | false | false   |
| false | true  | false   |
| true  | false | false   |
| true  | true  | true    |

✅ Таблица истинности для оператора `or`:

|   a   |   b   | a or b |
|:-----:|:-----:|:-------:|
| false | false | false   |
| false | true  | true    |
| true  | false | true    |
| true  | true  | true    |

✅ Таблица истинности для оператора `not`:

|   a   | not a |
|:-----:|:-----:|
| true  | false |
| false | true  |

**Приоритет:** `not` → `and` → `or` → используйте скобки для ясности!

## Циклы

### `for` — для каждого элемента
```python
# По списку
fruits = ['яблоко', 'банан']
for fruit in fruits:
    print(fruit)

# По диапазону
for i in range(1, 6):     # 1, 2, 3, 4, 5
    print(i)

# Обратный порядок
for i in reversed(range(1, 6)):
    print(i)              # 5, 4, 3, 2, 1
```

> ⚠️ `range(a, b)` включает `a`, исключает `b`

# Django
## Что такое Django?
**Django** — высокоуровневый Python-фреймворк с открытым кодом для быстрой и безопасной разработки веб-приложений. "Для перфекционистов с дедлайнами."

### Архитектура MVT
Django использует шаблон **Model-View-Template**:

| Компонент | Описание |
|-----------|----------|
| **Model** | Работа с БД, структура данных |
| **View** | Логика обработки запросов, что показать |
| **Template** | HTML-шаблоны, внешний вид |

> 💡 Она разделяет внутреннюю логику работы сайта, внешний вид страниц для пользователя и реакции веб-сервиса на внешние воздействия.

## Создание проекта

```bash
django-admin startproject myproject .
```
> ⚠️ **Важно!** Точка в конце создает проект в текущей папке без лишней вложенности.

### Структура проекта
```
myproject/
├── manage.py          # Главная утилита управления
├── myproject/         # Внутренняя папка настроек
│   ├── __init__.py    # Маркер Python-пакета
│   ├── settings.py    # Настройки проекта (БД, приложения, пути)
│   ├── urls.py        # Главный файл маршрутизации
│   └── wsgi.py        # Точка входа для сервера
└── myenv/             # Виртуальное окружение (не часть проекта)
```

### Запуск сервера
```bash
python manage.py runserver
```
Сервер доступен по адресу `http://127.0.0.1:8000/`. Остановка: `Ctrl+C`.

## Представления (Views)

Файл `views.py` содержит логику обработки запросов.

```python
from django.http import HttpResponse
from django.shortcuts import render

# Простое представление
def simple_view(request):
    return HttpResponse("Привет, мир!")

# Представление с шаблоном
def home_view(request):
    context = {
        'user_name': 'Соня',
        'weather': 'Пасмурно'
    }
    return render(request, 'home.html', context)
```

> 🎯 **render()** — функция-сокращение для отрисовки шаблона с контекстом.

## Маршрутизация (URLs)

### Главный `urls.py` проекта
```python
from django.contrib import admin
from django.urls import path, include
from myapp import views

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', views.home_view, name='home'),
    path('about/', views.about_view, name='about'),
    path('blog/', include('blog.urls')),  # Подключение URL приложения
]
```

### Функция `path()`
```python
path(route, view, name=None)
```
| Параметр | Описание | Пример |
|----------|----------|--------|
| `route` | URL-адрес | `'about/'`, `''` (главная) |
| `view` | Функция-обработчик | `views.about_view` |
| `name` | Имя для ссылок в шаблонах | `'about'` |

## Шаблоны (Templates)

### Настройка поиска шаблонов
В `settings.py` добавьте путь к папке `templates`:

```python
import os

TEMPLATES = [
    {
        # ...
        'DIRS': [os.path.join(BASE_DIR, 'templates')],
        # ...
    },
]
```

### Синтаксис шаблонов

**Переменные** `{{ variable }}` — вывод данных из контекста:
```html
<h1>Привет, {{ user_name }}!</h1>
<p>Сегодня {{ weather }}</p>
```

**Контекст** передается через `render()`:
```python
return render(request, 'index.html', {
    'user_name': 'Соня',
    'weather': 'Пасмурно'
})
```

**Теги** `{% tag %}` — управляющая логика:
```html
{% if user.is_authenticated %}
    <p>Добро пожаловать, {{ user.name }}</p>
{% else %}
    <a href="/login">Войти</a>
{% endif %}
```

## Статические файлы

### Настройки в `settings.py`
```python
STATIC_URL = '/static/'  # URL-префикс для доступа
STATICFILES_DIRS = [BASE_DIR / 'static']  # Папки со статикой проекта
```

### Где хранить статику?

**1. Внутри приложения** (для файлов конкретного приложения):
```
myapp/static/myapp/css/style.css
```

**2. На уровне проекта** (для общих файлов):
```
static/
├── css/
├── js/
└── images/
```

### Подключение в шаблоне
```html
{% load static %}

<link rel="stylesheet" href="{% static 'css/style.css' %}">
<script src="{% static 'js/script.js' %}"></script>
<img src="{% static 'images/logo.png' %}" alt="Logo">
```

> 🔄 Всегда используйте `{% load static %}` в начале шаблона перед подключением файлов.

## Модели в Django 
### Что такое модель?

**Модель** — это Python-класс, который описывает структуру таблицы в базе данных. Наследуется от django.db.models.Model.

```python
from django.db import models

class Product(models.Model):
    name = models.CharField(max_length=100)
    price = models.DecimalField(max_digits=10, decimal_places=2)
```

> 📦 Простая аналогия: Класс = таблица, атрибуты = столбцы, экземпляр = строка.

### Миграции
```bash
# После создания/изменения моделей:
python manage.py makemigrations    # создаем миграции
python manage.py migrate           # применяем к БД
python manage.py createsuperuser   # создание админа, для в входа в админку
```
> ⚠️ Важно: Команда **createsuperuser** работает только после применения миграций (migrate), так как создает записи в таблицах аутентификации.



### Файл `admin.py`
```python
from django.contrib import admin
from .models import Product

# Чтобы сделать модель доступной для управления через админку, ее необходимо зарегистрировать.

admin.site.register(Product) #Простейший способ регистрации модели
```
Чтобы сделать админку доступной по определенному URL, нужно настроить маршрутизацию в главном файле urls.py вашего проекта. 
```python
path('admin/', admin.site.urls), # Эта строка включает URL админки
```
### Что получилось?
 В админке Django появится: Раздел Products (Товары)

Теперь через админку можно:
- ➕ Создавать новые товары
- ✏️ Редактировать существующие
- 🔍 Просматривать список
- 🗑️ Удалять товары

## Админка в Django
### Вход в Админку
1. Запустите сервер разработки: python manage.py runserver
2. Откройте в браузере URL, который вы настроили для админки (обычно http://127.0.0.1:8000/admin/).
3. Вы увидите страницу входа. Введите имя пользователя и пароль суперпользователя, которые вы только что создали.
4. После успешного входа вы попадете на главную страницу админ-панели.
